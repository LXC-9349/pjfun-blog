## 我的信息焦虑症

2023 年我加的书签超过 2000 个，Instapaper 里标记"稍后阅读"的文章 400 多篇，印象笔记里存了 700 多条笔记，买了 8 个付费 newsletter——但是，当我想找一个"我记得看过"的信息时，经常花十几分钟找不到。

这不是储备不足，这是组织方式有问题。我花了很多精力收集信息，但没有花任何精力去**整理和内化**它们。收集本身给我一种"学到了"的错觉，实际上知识只是流过了我的大脑，没有留下任何痕迹。

## 问题出在"分类思维"

大多数人整理信息的方式是按主题分类——"前端"、"后端"、"AI"、"项目管理"——每个文件夹放对应的文章。

这个方式的问题在于：**一个信息可以有多个归属**。一篇关于"用 AI 做代码审查"的文章，应该放在"AI"还是"代码质量"还是"工程效率"？你怎么选？

而且，分类是静态的。你的认知框架在不断变化，去年的某个分类，今年看可能已经不适用了。但谁有精力去重构几千条笔记的分类呢？

## PARA 方法：打破分类的困局

我用的方法是 Tiago Forte 提出的 PARA 框架。它不按主题分类，而是按**"可操作性"**分类：

```
📁 项目（Projects）
  当前在做的具体任务，有截止日期和目标

📁 领域（Areas） 
  需要长期维护的职责，没有明确的终点

📁 资源（Resources）
  未来可能有用的主题资料

📁 归档（Archives）
  已完成或不再活跃的其他三项内容
```

### 项目文件夹

项目是所有笔记系统的核心。一个"项目"必须满足三个条件：
1. 有一个具体的目标
2. 有明确的截止日期或完成标准
3. 你当前正在做它

我目前的项目列表：
```
📁 项目/
  ├── 博客改版（Vite + Vue3 升级）
  ├── 2026 面试准备
  ├── 读书会组织方案
  ├── 自动化部署脚本
  └── 个人网站 SEO 优化
```

每个项目有自己的文件夹，包含相关的笔记、文档、参考资料。项目完成了，整个文件夹移到"归档"。

**项目驱动的核心理念**：只有当你正在做一件事时，与之相关的信息才值得被整理。那些"以后用得到"的资料，让它在"资源"里待着就好，不需要精细分类。

### 领域文件夹

领域是你生活中需要长期维护的责任范围。没有截止日期——你永远不会"完成"一个领域。

```
📁 领域/
  ├── 健康（运动、饮食、睡眠规划）
  ├── 财务（预算、投资、保险）
  ├── 技术成长（学习计划、技能树）
  ├── 关系（家人、朋友、同事）
  └── 写作（更新频率、主题规划）
```

领域的笔记通常是"参考性质"的——比如你的运动计划、常用的预算模板、年度学习目标。

### 资源文件夹

这是之前 "按主题分类" 的遗产。但有一个关键区别：**资源不需要被整理**。你只需要把感兴趣的链接、文章、笔记扔进去，贴一个标签或者加个关键词就行。不要花时间分类——反正你也不会经常回来翻。

```
📁 资源/
  ├── AI 工具记录
  ├── 好的设计案例
  └── 旅行攻略
```

### 归档

这是 PARA 最有价值的特性。任何完成的项目、过期的领域、不再相关的资源——全部移到"归档"。

删除很恐怖，归档很安心。归档让你能**毫无心理负担地开始新项目**——因为你不会丢失旧信息，但它也不会在你面前碍眼。

## 收集与处理：两周规则

光有结构不够，还得有处理流程。我的原则很简单：

**新信息进来后，问自己两个问题：**
1. 这和我当前的项目有关吗？
2. 没有的话，它属于哪个领域？

回答不了这两个问题的信息，90% 不值得保存。

我还有一个"两周规则"：新保存的东西如果两周内没被引用或使用过一次，就删掉。这个规则残忍但有效——它逼迫我消化信息而不是囤积信息。

## 工具选择和实操配置

我推荐 Obsidian，原因很简单——它基于 Markdown 文件，没有厂商锁定，而且双链功能设计得好。

### 我的工作流

```markdown
# 每日笔记模板

## 📌 今天最重要的三件事
1. 
2. 
3. 

## 📝 想法/记录
- [随时记录的想法]

## 📖 学到的东西
- [新知 + 来源]

## ⏭️ 未完成项/明天
- 
```

这种"以日笔记为入口"的方式降低了我记笔记的门槛。不需要找到"正确"的文件夹，打开每日笔记直接写，之后有时间了再整理。

### 整理节奏

我不实时整理知识。实时整理太花精力，而且很多信息在当下看不出来有没有价值。

我的节奏是**每周日晚上花 30 分钟做一次清理**：
1. 浏览本周收集的内容
2. 把有长期价值的内容链接到项目/领域
3. 删除不需要的
4. 把每日笔记中散落的有价值信息提取出来

## 信息源的节制

这是知识管理中容易被忽视的一环——输入质量决定了整理效果的上限。

我做了这些调整：
- **退订了 80% 的 newsletter**：只留下 2-3 个真正每期都看的
- **限制 RSS 订阅**：控制在 15 个以内，超过就替换
- **关掉了大部分通知**：手机上除了即时通讯 App，其他通知全关
- **每周一天"断网日"**：不主动消费信息，只处理已有的笔记

信息源减少后，我发现根本不需要花那么多时间整理——因为进来的东西本来就不多了。

## 输出倒逼输入

知识管理的终点不是"有很多笔记"，而是"能产生点什么"。

我不再追求"读完一本书写一篇完美的总结"。现在我的流程是：

1. 读到一个有意思的观点
2. 在自己的笔记里写下理解和联想（一两个句子）
3. 如果过了两周这个观点我还能想起来——它真正属于我了，可以考虑写成一篇文章

这个"两周过滤器"帮我筛掉了 90% 的过眼云烟。留下的 10% 才是真正对我有影响的知识。

## 写在最后

知识管理不是为了管理知识，是为了让你知道得更少、思考得更好。

囤积信息给人安全感，但真正的安全感来自——你确信自己能在需要的时候找到所需的信息。

在这个前提下，少就是多。
