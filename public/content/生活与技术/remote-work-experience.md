## 为什么写这个

2023 年开始远程工作，到现在三年了。前三个月差点崩溃——效率低、沟通不畅、感觉和团队脱节。后来慢慢找到了节奏，到现在我已经不太能想象回到每天通勤两小时去办公室的日子了。

这篇文章不是要说服谁远程工作更好——它适合你。如果你已经在远程或正在考虑远程，这里的经验可能会帮你少走一些弯路。

## 沟通：文字比语音更适合异步

远程工作最大的挑战不是技术问题，是沟通。

### 能写就写，别什么都开会

有一个很普遍的误区：远程工作需要更多的会议来"对齐"。

实际上，大多数信息交流**不需要实时**。对于一个设计决策或问题讨论，写一段文档或消息比拉一个会议高效得多：

```
开会（30 分钟）：
- 花 5 分钟确认都有谁在线
- 花 10 分钟讲背景
- 花 10 分钟讨论
- 花 5 分钟总结
= 30 分钟，而且没有记录

写文档（15 分钟）：
- 花 10 分钟写背景和方案
- 5 分钟写请求大家反馈
= 15 分钟，永久记录，可异步查阅
```

异步文字沟通还有一个好处：**它会迫使你把思路写清楚**。很多你以为想清楚了的东西，在写的过程中会发现逻辑其实没闭合。

### 用"状态"替代"在吗"

```text
❌ "在吗？"
✅ "关于 X 方案，我遇到了 Y 问题，尝试了 Z 方案没解决。
   需要你的建议。不着急，今天内回复都可以。"
```

第一种消息的问题是：对方看到后必须回复，但不知道你要说什么——打乱了对方的工作流。

第二种消息包含了所有上下文，对方可以在方便的时候一次性处理完。

### 两条沟通原则

1. **给上下文**：不说"看看这个"，说"这个 PR 修复了登录页在 iOS 上的样式问题，改动不大，主要改了 padding 和字体"

2. **给预期**：不说"这个方案你评估一下"，说"这个方案你评估一下，目标是这周五前定下来"

预期管理在远程工作中比在办公室重要得多——因为没有面对面的状态可见性。

## 专注：保护你的"大块时间"

远程工作最大的好处是可以专注。但也最容易失去专注——因为缺少了办公室的"大家都在工作"的心理暗示。

### 时间分块

我的做法是把一天分成三个时段：

```
上午 9:00-12:00 —— 深度工作
  写代码、方案设计、代码审查（需要大块专注力的工作）
  手机静音，不看消息

下午 13:00-16:00 —— 协作时间
  开会、回复消息、Pair Programming
  
下午 16:00-18:00 —— 收尾
  回邮件、写文档、整理 TODO（不需要太多脑力的工作）
```

这个节奏不是固定的——有紧急上线或者重大 Bug 时会调整。但日常维持这个节奏让我能保持稳定的产出。

关键原则：**把需要深度思考的工作安排在整块时间里，把这些时间当成不可打扰的。**

### 快速进入状态的技巧

```text
环境准备：
- 固定的工作区域（不要在床上工作）
- 工作时穿工作服（不一定是正装，但不要穿睡衣）
- 开始工作前先做五分钟的计划

切换信号：
- 用不同的音乐/白噪音标记工作状态
- 番茄钟（25 分钟工作 + 5 分钟休息）
- 每天第一件事是写当天的 TODO
```

### 对抗疲惫感

远程工作一个常见陷阱：**因为没有上下班的界限，最终变成了全天都在工作。**

```text
不好的模式：
- 一直坐在电脑前面
- 吃饭也在工位上
- 晚上还在回消息
- 周末也忍不住看代码

更好的模式：
- 每天固定工作时间（我的：9:00-18:00）
- 到点了就关电脑（物理关闭，不仅仅是合上盖子）
- 工作日结束时写下明天的 TODO
- 至少一小时不看屏幕
```

我现在到 18:00 就关掉工作电脑。有时会忍不住用个人电脑看代码——但我强制自己在关掉后离开书房去做别的事情。

"停止了"是一个人需要长期保持的生产力的关键。如果你一直在工作，既没有生活，工作质量也会下降。

## 边界：工作与生活的分离

### 物理边界

如果你住的地方没有多余房间当书房，至少要做到：**工作桌不用来做其他事情**。

如果你在那张桌子吃饭、打游戏、看电影，你的大脑在进入那个空间时不会自动切换成"工作模式"。

### 心理边界

远程工作中，你需要主动告诉团队"我不在线了"：

- Slack/微信状态设置为"离开"
- 在日历上标记为"忙碌"
- 团队有事情在非工作时间找到你时，如果不紧急，第二天再回复

这种边界不是自私——是可持续工作的前提。你的同事也希望你有边界，好让他们也有。

### 社交联系

远程工作最大的隐性成本是社交减少。没有午餐时的闲聊、茶水间的偶遇、下班后的聚餐。

我采取的方法：
- 每周约同事视频喝咖啡（15 分钟，不聊工作）
- 参加线上技术社区的活动
- 定期（每月）参加线下的 meetup

人的社交需求是真的。你可能短时间内觉得"一个人工作真好"，但长期缺乏社交接触会影响状态。

## 工具：简单比复杂好

```
沟通：Slack/微信 → 一个作为主力
文档：Notion/飞书 → 团队统一用一套
代码协作：GitHub → PR + Code Review
视频：Zoom/腾讯会议 → 稳定比功能重要
TODO：GitHub Issues 或 Notion → 不要超过一个工具
```

工具数量控制在最低限度。用越多的工具，信息越分散，切换成本越高。如果一个工具团队的活跃度很低，主动提议停用它，把工作流移到主力工具中。

## 写在最后

远程工作不是适合所有人。有些人需要办公室的社交和结构感才能高效工作。如果你试了几个月觉得不适应，不需要强迫自己——回办公室没有任何不对的地方。

但如果你决定尝试，至少坚持三个月。第一个月会非常不适应，第二个月开始找到节奏，第三个月才能客观评价。
